# Модульная структура проекта

## Обзор

Монолитный файл `page.tsx` (1973 строки) был разбит на модульную структуру для улучшения поддерживаемости и масштабируемости.

## Структура папок

```
app/
├── types/
│   └── index.ts                 # Все TypeScript типы и интерфейсы
├── constants/
│   ├── mockData.ts              # Mock данные (части, слоты, сервисы, клиенты)
│   └── initialState.ts          # Начальное состояние приложения
├── hooks/
│   ├── useSystemState.ts        # Хук для управления состоянием системы
│   ├── useToast.ts              # Хук для уведомлений
│   └── index.ts                 # Экспорты хуков
├── utils/
│   ├── handlers.ts              # Обработчики событий
│   └── helpers.ts               # Вспомогательные функции
├── components/
│   ├── shared/
│   │   ├── Toast.tsx            # Компонент уведомлений
│   │   ├── Header.tsx           # Шапка приложения
│   │   └── index.ts
│   ├── client/
│   │   ├── ClientMobileApp.tsx  # Главный компонент клиентского приложения
│   │   ├── FormView.tsx         # Форма создания заявки
│   │   ├── TrackingView.tsx     # Отслеживание статуса заказа
│   │   └── index.ts
│   ├── admin/
│   │   ├── AdminWebApp.tsx      # Главный компонент админ-панели
│   │   ├── IncomingColumn.tsx   # Колонка входящих заявок
│   │   ├── PartsReviewColumn.tsx # Колонка проверки запчастей
│   │   ├── ScheduleColumn.tsx   # Колонка расписания
│   │   ├── ScheduledColumn.tsx  # Колонка запланированных
│   │   ├── CompletedColumn.tsx  # Колонка завершенных
│   │   ├── ScheduleView.tsx     # Вид расписания
│   │   ├── ClientsView.tsx      # Вид клиентов
│   │   ├── RejectionModal.tsx   # Модальное окно отклонения
│   │   └── index.ts
│   └── tech/
│       ├── TechTabletApp.tsx    # Главный компонент планшета механика
│       ├── TechWorkspace.tsx    # Рабочее пространство механика
│       ├── AdditionalTaskModal.tsx # Модальное окно доп. задач
│       └── index.ts
├── page.tsx                     # Оригинальный монолит (сохранен)
└── page-new.tsx                 # Новая модульная версия

## Преимущества новой структуры

### 1. Разделение ответственности
- **types/** - Все типы в одном месте
- **constants/** - Константы и mock данные отдельно
- **hooks/** - Переиспользуемая логика состояния
- **utils/** - Вспомогательные функции
- **components/** - UI компоненты по ролям

### 2. Улучшенная поддерживаемость
- Каждый файл < 300 строк
- Легко найти нужный код
- Простое тестирование отдельных модулей

### 3. Переиспользование кода
- Хуки можно использовать в других компонентах
- Типы доступны во всем приложении
- Утилиты можно расширять

### 4. Масштабируемость
- Легко добавлять новые компоненты
- Простое расширение функциональности
- Удобная командная работа

## Ключевые модули

### Хуки

**useSystemState** - Управление глобальным состоянием
```typescript
const { state, setState, updateState, reset } = useSystemState();
```

**useToast** - Управление уведомлениями
```typescript
const { toast, showToast } = useToast();
```

### Утилиты

**handlers.ts** - Все обработчики событий в одном месте
```typescript
const handlers = createHandlers(state, updateState, showToast);
```

**helpers.ts** - Вспомогательные функции (например, getOpacity)

### Компоненты

**ClientMobileApp** - iOS приложение клиента
- FormView - Форма заявки
- TrackingView - Отслеживание статуса

**AdminWebApp** - Веб CRM администратора
- Kanban колонки для управления заявками
- Просмотр расписания и клиентов

**TechTabletApp** - iPad приложение механика
- Рабочее пространство с чек-листом
- Фотографирование при приемке
- Отчет о дополнительных проблемах

## Миграция

Оригинальный файл `page.tsx` сохранен для обратной совместимости.
Новая версия находится в `page-new.tsx`.

Для переключения на новую версию:
1. Переименуйте `page.tsx` в `page-old.tsx`
2. Переименуйте `page-new.tsx` в `page.tsx`
3. Запустите `npm run dev`
